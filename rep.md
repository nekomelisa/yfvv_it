# Тема 1. Алгоритми та обчислювальна складність

## 1.1 Основи структур даних і алгоритми

### 1.1.1 Поняття алгоритму. Часова та просторова складність
**Алгоритм** — послідовність точних інструкцій для виконання завдання. Властивості: **визначеність** (кожна інструкція чітка), **виконуваність**, **скінченність** (завершується за скінченну кількість кроків), **результативність**.

**Часова складність** — залежність часу виконання від розміру входу *n*, описується О-нотацією. **Просторова складність** — обсяг пам'яті (змінні, рекурсивні виклики, допоміжні структури).

Основні класи складності: **O(1)** — стала (доступ до елемента масиву); **O(log n)** — логарифмічна (бінарний пошук); **O(n)** — лінійна (пошук максимуму); **O(n²)** — квадратична (сортування вставками); **O(2ⁿ)** — експоненційна (задача комівояжера).

### 1.1.2 Абстрактні типи даних (АТД)
**АТД** — модель, що визначає логічну структуру даних і набір операцій, без деталей реалізації.

- **Стек** — принцип **LIFO** (останній прийшов — перший вийшов); операції push, pop, peek.
- **Черга** — принцип **FIFO** (перший прийшов — перший вийшов); enqueue, dequeue.
- **Черга з пріоритетами** — елементи з вищим пріоритетом обробляються першими.
- **Список** — впорядкована послідовність елементів з доступом за індексом.
- **Вектор** — динамічний масив (автоматично зростає, доступ за індексом O(1)).
- **Словник (хеш-таблиця)** — пари «ключ–значення», унікальні ключі, доступ O(1).
- **Множина** — унікальні неупорядковані елементи; операції об'єднання, перетину, різниці.
- **Мультимножина** — як множина, але з повторами (враховує кратність).

### 1.1.3 Складність операцій над структурами
| Структура | Доступ | Дод. на початок | Дод. в кінець | Пошук |
|---|---|---|---|---|
| Кортеж (незмінний) | O(1) | — | — | O(n) |
| Множина | O(n) | O(1) | — | O(1) |
| Словник | O(1) | O(1) | — | O(1) |
| Однозв'язний список | O(n) | O(1) | O(n) | O(n) |
| Двозв'язний список | O(n) | O(1) | O(1) | O(n) |

**Кортеж** — незмінна послідовність. **Однозв'язний список** — вузол зберігає значення + посилання на наступний; **двозв'язний** — ще й на попередній.

### 1.1.4 Пошук і сортування
**Пошук:** лінійний (перебір по черзі, O(n)); бінарний (лише у відсортованому масиві, поділ навпіл, O(log n)).

**Прості сортування:** вибором, вставками, обмінами (бульбашкове) — O(n²).
**Удосконалені:** деревом (Heap Sort), Шелла, швидке (Quick Sort).

| Алгоритм | Найкращий | Середній | Найгірший |
|---|---|---|---|
| Лінійний пошук | O(1) | O(n) | O(n) |
| Бінарний пошук | O(1) | O(log n) | O(log n) |
| Вибором | O(n²) | O(n²) | O(n²) |
| Вставками / бульбашкове | O(n) | O(n²) | O(n²) |
| Деревом (Heap) | O(n log n) | O(n log n) | O(n log n) |
| Швидке (Quick) | O(n log n) | O(n log n) | O(n²) |

### 1.1.5 Алгоритми на графах
- **BFS (пошук вшир)** — обхід рівнями через чергу; знаходить найкоротший шлях у **незважених** графах. O(V+E).
- **DFS (пошук вглиб)** — обхід максимально в глибину через стек/рекурсію. O(V+E).
- **Зв'язні компоненти** — DFS/BFS з кожної непоміченої вершини. O(V+E).
- **Кістякове (остовне) дерево** — з'єднує всі вершини без циклів з мінімальною вагою: алгоритми **Пріма** (нарощує від однієї вершини) і **Крускала** (додає ребра за зростанням ваги).
- **Найкоротші шляхи:** **Дейкстра** (додатні ваги), **Беллмана–Форда** (допускає від'ємні ваги), **A\*** (з евристикою).

## 1.2 Стратегії розроблення алгоритмів

### 1.2.1 Розділяй та володарюй
Поділ задачі на менші підзадачі того ж типу → рекурсивне розв'язання → об'єднання результатів. Етапи: **Divide → Conquer → Combine**. Приклади: сортування злиттям, швидке сортування, бінарний пошук, FFT.

### 1.2.2 Балансування
Підтримання рівномірного розподілу елементів/обчислень для ефективних операцій. Приклади: **збалансовані дерева** (AVL, червоно-чорне — операції O(log n)), балансування навантаження (Load Balancing), work-stealing.

### 1.2.3 Динамічне програмування (DP)
Метод для задач із **підзадачами, що перекриваються**: кожну підзадачу розв'язують один раз і зберігають результат. Підходи: **мемоізація** (зверху вниз) і **табуляція** (знизу вгору). Приклади: числа Фібоначчі, задача про рюкзак (0/1), найдовша спільна підпослідовність (LCS), відстань редагування.

### 1.2.4 Оцінка складності стратегій
Сортування злиттям — O(n log n); операції у AVL/червоно-чорному дереві — O(log n); рюкзак 0/1 — O(n·W); LCS — O(m·n).

## 1.3 Моделі обчислень

### 1.3.1 Імперативний та декларативний підходи
**Імперативний** — описує **як** досягти результату (покрокові інструкції; C, Java, Python). **Декларативний** — описує **що** треба отримати, без деталей процесу (SQL, HTML, Haskell).

### 1.3.2 Розв'язність. Проблема зупинки
- **Розв'язні (decidable)** — існує алгоритм, що завжди завершується з правильною відповіддю (перевірка простоти, алгоритм Евкліда).
- **Напіврозв'язні (semidecidable)** — алгоритм дає відповідь лише для позитивних випадків; для негативних може працювати нескінченно.
- **Нерозв'язні (undecidable)** — алгоритму не існує для всіх входів.
- **Проблема зупинки (Halting Problem)** — класичний приклад нерозв'язної задачі (Тюрінг, 1936): не існує універсального алгоритму, який для будь-якої програми P і входу I визначить, чи зупиниться P. Доводиться від протилежного.

# Тема 2. Архітектура комп'ютера

## 2.1 Функції бінарної логіки
- **AND (І)** — 1, лише коли обидва входи 1.
- **OR (АБО)** — 1, коли хоча б один вхід 1.
- **NOT (НЕ)** — інверсія.
- **XOR (виключне АБО)** — 1, коли входи різні.
- **NAND (І-НЕ)** — інверсія AND (0 лише коли обидва 1).
- **NOR (АБО-НЕ)** — інверсія OR (1 лише коли обидва 0).

## 2.2 Подання даних на рівні машин

### 2.2.1 Системи числення та цілі числа
**Позиційна система:** значення цифри залежить від її позиції та основи. **Двійкова** (основа 2), **вісімкова** (8; одна цифра = 3 біти), **шістнадцяткова** (16; одна цифра = 4 біти).

**Беззнаковий код** — лише додатні числа, усі біти під значення; для *n* бітів діапазон 0…2ⁿ−1.
**Доповнювальний код (two's complement)** — додатні й від'ємні: старший біт = знак; діапазон −2ⁿ⁻¹…2ⁿ⁻¹−1; єдине представлення нуля. Віднімання = додавання доповнювального коду; переповнення = вихід за межі діапазону.

### 2.2.2 Дійсні числа (плаваюча кома)
Формат **IEEE 754**: число = мантиса × основа^показник. Складові: **знаковий біт**, **експонента (показник)** зі зміщенням (біас 127 для 32-біт, 1023 для 64-біт), **мантиса**. Нормалізоване число має неявну «1».
**Проблеми:** похибки округлення, втрата точності при різних порядках, порушення асоціативності, переповнення/недоповнення, значення Infinity та NaN.

## 2.3 Введення-виведення. Шина
**Пристрої введення** (клавіатура, миша), **виведення** (монітор, принтер), **введення-виведення** (диски, мережеві адаптери). Способи взаємодії: програмоване, за перериваннями, **DMA** (прямий доступ до пам'яті).

**Шина** — набір ліній для обміну даними. Види: **шина даних** (ширина = скільки бітів за раз), **адресна шина** (визначає обсяг адресованої пам'яті), **шина керування** (сигнали синхронізації, переривань). Стандарти: PCI/PCIe, USB, SATA.

## 2.4 Функціональна організація

### 2.4.1 Фон Неймана та Гарвардська архітектури
| | Фон Неймана | Гарвардська |
|---|---|---|
| Пам'ять | Спільна для даних і команд | Роздільна |
| Шини | Одна | Окремі для даних і команд |
| Швидкість | Повільніше (конфлікт доступу) | Швидше (паралельний доступ) |
| Застосування | Універсальні комп'ютери | Мікроконтролери, DSP, GPU |

**Фон Неймана** (1945): єдина пам'ять, послідовне виконання команд; недолік — «вузьке місце» пам'яті.

### 2.4.2 Ієрархія пам'яті. CPU
Від найшвидшої/найдорожчої до повільної/дешевої: **регістри → кеш (L1/L2/L3) → оперативна пам'ять (RAM) → зовнішня пам'ять (HDD/SSD)**. Часто вживані дані — ближче до процесора.

**CPU** складається з: **АЛП (ALU)** — арифметико-логічні операції; **пристрій керування (CU)** — інтерпретація команд; **регістри**; **кеш**. Цикл роботи: **Fetch (завантаження) → Decode (декодування) → Execute (виконання)**. Характеристики: тактова частота, кількість ядер, обсяг кешу, розрядність (32/64-біт).

### 2.4.3 Периферійні пристрої
Зовнішні/внутрішні компоненти для введення, виведення, зберігання, зв'язку. Характеристики: інтерфейс (USB, HDMI, Bluetooth), швидкість, сумісність, продуктивність, енергоспоживання.

# Тема 3. Бази та сховища даних

## 3.1 Ключі та нормалізація
Нормалізація зменшує надлишковість і запобігає аномаліям.
- **1NF** — усі значення **атомарні** (неділимі), немає повторюваних груп.
- **2NF** — 1NF + неключові атрибути залежать від **усього** первинного ключа (актуально для складених ключів).
- **3NF** — 2NF + немає **транзитивних** залежностей (неключовий атрибут не залежить від іншого неключового).
- **BCNF** — 3NF + кожен **детермінант є ключем**.

## 3.2 Основні концепції СУБД
- **Модель даних** — абстрактна структура організації даних (реляційна, ієрархічна, мережева, об'єктна, документо-орієнтована, ключ-значення).
- **Мова запитів** — засіб взаємодії з БД (SQL, NoSQL, XQuery, Cypher, SPARQL).
- **Транзакція** — послідовність операцій як єдине ціле (усе або нічого): Start → Execution → Commit / Rollback.
- **ACID:** **Atomicity** (атомарність — усе або нічого), **Consistency** (узгодженість — цілісність зберігається), **Isolation** (ізольованість — транзакції не впливають одна на одну), **Durability** (надійність — зміни зберігаються після збою).
- **Індексування** — структура для прискорення пошуку (B-дерева, хеші). Типи: первинний, унікальний, кластеризований, некластеризований, складений. Мінус — сповільнення запису.
- **Резервне копіювання/відновлення** — повне, інкрементне, диференційне, дзеркальне, журнальне.
- **Розподіленість** — частини даних на різних вузлах (шардинг горизонтальний / партиціонування вертикальне). **Реплікація** — копії даних (синхронна/асинхронна; master-slave; multi-master).
- **Безпека даних** — конфіденційність, цілісність, доступність, автентичність, відстежуваність; методи: шифрування, контроль доступу, автентифікація.

## 3.3 Моделювання даних
Три рівні: **концептуальна** (бізнес-сутності та зв'язки, без прив'язки до СУБД) → **логічна** (атрибути, ключі PK/FK, нормалізація) → **фізична** (таблиці, типи даних, індекси для конкретної СУБД).

**ER-модель (сутність–зв'язок):** компоненти — **сутність** (об'єкт), **атрибут** (властивість), **зв'язок** (1:1, 1:N, N:M), **ключ**, **кардинальність**, слабка сутність. Нотації: **Чена** (прямокутники/ромби/овали), **«вороняча лапа»** (Crow's Foot — компактна), **IDEF1X**.

## 3.4 Реляційні БД. DBMS
**Реляційна БД** (Кодд, 1970): дані у **таблицях (відношеннях)** — рядки (кортежі) + стовпці (атрибути). Ключі: **первинний** (унікальність), **зовнішній** (зв'язки). Властивості таблиці: атомарність комірок, однорідність стовпців, унікальні імена стовпців, відсутність однакових рядків, довільний порядок.

**DBMS (СУБД)** — ПЗ для створення й керування БД: зберігання, маніпуляція, контроль доступу, захист, багатокористувацький доступ.

## 3.5 SQL та його підмови
- **DDL** (Data Definition) — структура: CREATE, ALTER, DROP, TRUNCATE.
- **DML** (Data Manipulation) — дані: SELECT, INSERT, UPDATE, DELETE.
- **DCL** (Data Control) — доступ: GRANT, REVOKE.
- **TCL** (Transaction Control) — транзакції: COMMIT, ROLLBACK, SAVEPOINT.

## 3.6 Реляційна алгебра
Унарні та бінарні операції над відношеннями:
- **Відбір (selection)** — рядки за умовою.
- **Проєкція (projection)** — вибрані стовпці.
- **Об'єднання (union)** — унікальні рядки двох таблиць однакової структури.
- **Перетин (intersection)** — спільні рядки.
- **Різниця (difference)** — рядки першої, яких немає в другій.
- **Декартів добуток (cartesian product)** — кожен рядок з кожним.
- **З'єднання (join)** — рядки за спільним атрибутом.
- **Ділення (division)** — рядки, що відповідають усім значенням іншої таблиці.

# Тема 4. Інженерія систем і програмного забезпечення

## 4.1 Складні та великі системи

### 4.1.1 Властивості систем
- **Емерджентність** — у системи в цілому виникають нові властивості, не притаманні окремим елементам (поведінка ≠ сумі частин).
- **Адитивність** — властивості системи = сумі властивостей компонентів (характерна для простих систем).
- **Еквіфінальність** — здатність досягати однакового кінцевого стану з різних початкових умов і шляхів.

### 4.1.2 Відкриті та закриті системи
**Закрита** — ізольована від середовища. **Відкрита** — взаємодіє з середовищем (інформаційно, енергетично, матеріально). Класифікація: за призначенням (виробничі, керуючі, обслуговуючі), походженням, видом елементів, способом організації.

### 4.1.3 Складні vs великі системи
**Складна** — складність через численні взаємодії між компонентами (напр. система управління польотами). **Велика** — характеризується великою кількістю користувачів/даних/функцій, моделювання ускладнене розмірністю (напр. Amazon). Дослідження великих систем: **композиція** (від елементів до системи) та **декомпозиція**.

## 4.2 Моделі систем

### 4.2.1 Склад/структура. Чорна та біла скринька
**Система** — сукупність взаємопов'язаних елементів, об'єднаних спільною метою. **Склад** — елементи + зв'язки.
- **Модель «чорної скриньки»** — відомі лише входи й виходи, внутрішня будова невідома (тестування за вимогами, без коду).
- **Модель «білої скриньки»** — відома внутрішня структура і код (перевірка логіки, безпеки).

### 4.2.2 Види моделей
**Концептуальна** — змістовна модель предметної області (поняття, причинно-наслідкові зв'язки). **Математична** — формальний опис через рівняння/формули. **Комп'ютерна** — реалізація моделі у програмному середовищі для симуляції.

### 4.2.3 Зв'язок системи та моделі. Гомоморфізм
**Модель** — спрощене формалізоване представлення системи. **Ізоморфізм** — бієктивна відповідність, що зберігає всі структури й відносини (взаємно однозначно). **Гомоморфізм** — відповідність зі спрощенням: модель зберігає лише суттєві відносини (втрачаючи частину деталей).

## 4.3 Інформаційні системи

### 4.3.1 Поняття та класифікація
**Інформаційна система (ІС)** — сукупність організаційних і технічних засобів для збереження й обробки інформації для потреб користувачів. Цілі: автоматизація бізнес-процесів, підтримка прийняття рішень, підвищення конкурентоспроможності. Класифікація за функціональністю: **TPS** (операційні), **DSS** (аналітичні), **KMS** (управління знаннями); за масштабом і сферою застосування.

### 4.3.2 Забезпечення ІС
Сім видів: **організаційне, інформаційне, математичне, програмне, технічне, лінгвістичне, методичне, правове**.

## 4.4 Аналіз вимог
**Вимога** — специфікація того, що має бути реалізовано.
- **Бізнес-вимоги** — високорівневі цілі організації.
- **Функціональні** — **що** система робить (конкретні дії, поведінка, реакції).
- **Нефункціональні** — **наскільки добре** працює (продуктивність, безпека, масштабованість, надійність).
- **Вимоги користувача** — фіксуються як **варіанти використання (use cases)** та **історії користувачів (user stories)**: «Як [роль], я хочу [дію], щоб [ціль]».
Джерела/методи збирання: інтерв'ю, анкетування, спостереження, аналіз документів, прототипування.

## 4.5 Проєктування ПЗ

### 4.5.1 Види проєктування
**Структурне** (Structural), **об'єктно-орієнтоване** (OOD), **функціональне** (Functional), **архітектурне** (Architectural), **інтерфейсне** (Interface Design).

### 4.5.2 Парадигми проєктування
Функціональна декомпозиція **згори донизу**; архітектура, **орієнтована на дані**; **об'єктно-орієнтований** аналіз і проєктування; **подієво-керована** архітектура.

### 4.5.3–4.5.4 UML
**Діаграми класів** — моделюють підсистеми, класи, зв'язки (ієрархія). **Діаграми послідовностей** та **комунікації** — сценарії реалізації варіантів використання (взаємодія об'єктів у часі).

### 4.5.5 Патерни проєктування
- **MVC** — розділення на Модель / Вигляд / Контролер.
- **Породжувальні:** Abstract Factory.
- **Структурні:** Facade (спрощений інтерфейс), Decorator (додавання поведінки), Flyweight (спільне використання об'єктів), Proxy (замісник).
- **Поведінкові:** Observer (спостерігач), Strategy (стратегія), Visitor (відвідувач), Chain of Responsibility (ланцюг відповідальності).

## 4.6 Реалізація ПЗ
- **Оформлення коду** — стиль, структурування, іменування змінних/класів.
- **Автоматична генерація коду** — за моделями/шаблонами.
- **Налагодження:** Breakpoints (точки зупинки), Variable Watch (спостереження за змінними), Console Output, Debugger, Code Analyzers.
- **Керування конфігурацією та версіями** — системи контролю версій (напр. Git).
- **CI/CD** — постійна інтеграція (автоматичне складання й тестування) / постійне впровадження.

## 4.7 Забезпечення якості
- **Тестування** — перевірка на дефекти. **Верифікація** — «чи правильно **побудували**?» (відповідність специфікації). **Валідація** — «чи побудували **правильний** продукт?» (відповідність потребам користувача).
- **Чорна скриня** — за вимогами, без коду; **біла скриня** — з доступом до коду (покриття операторів/гілок/шляхів).
- **Рівні тестування:** модульний → інтеграційний → системний → валідаційний (приймальний).
- **TDD (розробка через тестування)** — спершу пишуть тест, потім код, що його проходить.
- Додаткові техніки: інспекція коду, перевірка відповідності стандартам, оцінка зручності (UX), тестування продуктивності й масштабованості.

## 4.8 Командна робота та підходи до розробки

### 4.8.1 Класичні моделі
- **Каскадна (водоспадна, Waterfall)** — послідовні незворотні етапи; для стабільних вимог.
- **Ітеративна (ітераційна)** — повторення етапів із урахуванням зворотного зв'язку.
- **Інкрементна** — розробка частинами (інкрементами), кожна додає функціональність.

### 4.8.2 Промислові технології
- **RUP** — раціональний уніфікований процес; 4 фази: початок, розвиток, конструювання, передача.
- **MSF** — методологія Microsoft Solutions Framework.
- **Agile** — гнучка розробка на основі ітерацій та співпраці самоорганізованих команд (Agile-маніфест, 12 принципів).
- **Scrum** — фреймворк Agile: спринти, ролі (Product Owner, Scrum Master, команда).
- **XP (Extreme Programming)** — акцент на задоволення потреб клієнта; цінності: простота, комунікація, зворотний зв'язок, повага, сміливість.
- **Kanban** — візуалізація потоку робіт на дошці, обмеження WIP.

### 4.8.3–4.8.4 Команда та ЖЦ проєкту
Ролі та обов'язки в команді; переваги командної роботи й ризики співпраці. **Життєвий цикл ІТ-проєкту** — від рішення про створення до вилучення з експлуатації; основні етапи планування й виконання.

# Тема 9. Основи мов програмування

## 9.1 Сутність і види мов програмування

### 9.1.1 Клас, об'єкт, конструктор, деструктор, інтерфейс
- **Клас** — шаблон (група даних і методів для роботи з ними); абстракція.
- **Об'єкт** — конкретна реалізація (екземпляр) класу; має стан (змінні), методи (операції) та ім'я.
- **Конструктор** — спеціальний метод, що викликається автоматично при **створенні** об'єкта; ініціалізує його стан. Може бути перевантажений (типовий, копіювання).
- **Деструктор** — метод, що викликається при **знищенні** об'єкта; звільняє ресурси (пам'ять, файли, з'єднання).
- **Інтерфейс (протокол)** — опис методів, які об'єкт розуміє, без реалізації; засіб взаємодії незв'язаних об'єктів. **Реалізація** — конкретне тіло цих методів.

### 9.1.2 Базові концепції ООП
- **Абстракція** — виділення лише суттєвих властивостей об'єкта, відсторонення від несуттєвих деталей.
- **Інкапсуляція** — приховування внутрішньої будови об'єкта; доступ лише через публічний інтерфейс (API). Модифікатори доступу: **public, protected, private**, за замовчуванням.
- **Спадкування (наслідування)** — новий клас (підклас) успадковує властивості й методи базового (суперкласу); реалізує відношення «є» (is-a).
- **Поліморфізм** — однакове ім'я методу для об'єктів різних класів дає різну поведінку відповідно до реального похідного класу («багато форм»).

### 9.1.3 Зв'язки між класами
- **Асоціація** — загальний зв'язок взаємодії; об'єкти існують незалежно.
- **Агрегація** — «має частину», але частини можуть існувати окремо від цілого (відділ ↔ професор).
- **Композиція** — жорсткіша агрегація: частина не існує без цілого (автомобіль ↔ двигун).
- **Спадкування** — дочірній клас отримує властивості батьківського.
- **Залежність** — клас використовує інший як параметр/локальну змінну, але не зберігає його стан.
- **Реалізація** — клас реалізує інтерфейс (абстрактний клас).

### 9.1.4 Процедурне vs ООП
**Процедурне** — послідовність дій, поділ на процедури/функції (C, Pascal, Fortran); простіше й швидше, але обмежена гнучкість. **ООП** — програма як сукупність об'єктів; гнучкіше й масштабованіше, але потребує більше часу й абстракції.

## 9.2 Види програмування
- **Функціональне** — обчислення через **чисті функції** (без побічних ефектів), **незмінні дані**, рекурсія, функції вищого порядку (Haskell, Clojure, Scala, F#, Erlang, Lisp).
- **Логічне** — опис **фактів і правил**, а не послідовності дій; висновок через логічне слідування (Prolog); для експертних систем, NLP, ШІ.
- **Подієво-орієнтоване** — виконання визначається **подіями** (дії користувача, повідомлення, сигнали ОС); головний цикл: отримання події → обробка. Для GUI, серверів, ігор.
- **Реактивне** — потоки даних та їх автоматична обробка в реальному часі; асинхронність, патерн Observer (RxJava, RxJS).
- **Узагальнене (generic)** — опис алгоритмів і структур незалежно від конкретних типів даних (шаблони в C++, generics у Java).

## 9.3 Класифікація Флінна (моделі паралельних обчислень)
За потоками команд та даних (Флінн, 1966):
- **SISD** — один потік команд, один потік даних (класичний одноядерний процесор).
- **SIMD** — одна команда над багатьма даними (GPU, векторні процесори; обробка масивів, зображень).
- **MISD** — багато команд над одними даними (рідкісна, для відмовостійких систем).
- **MIMD** — багато команд над багатьма даними незалежно (багатоядерні процесори, кластери, суперкомп'ютери).

## 9.4 Трансляція та виконання
- **Компілятор** — перетворює **всю** програму на машинний код відразу; виконання швидше, помилки виявляє під час компіляції (C, C++, Rust, Go).
- **Інтерпретатор** — виконує код **порядково** без попередньої компіляції; легше налагоджувати, але повільніше (Python, Ruby, PHP, Perl). Може працювати в режимі REPL.
- **Компонувальник (linker)** — об'єднує об'єктні файли (.o/.obj) + бібліотеки у виконуваний файл, розв'язує зовнішні посилання. Компонування **статичне** (бібліотеки включені у файл) або **динамічне** (посилання на .dll/.so).

Цикл: редактор → компілятор → компонувальник → завантажувач (loader).

# Тема 10. Штучний інтелект

## 10.1 Фундаментальні поняття
- **Інтелектуальна система** — програмно-апаратний комплекс, що імітує функції інтелекту: аналіз даних, прийняття рішень, навчання, адаптація.
- **Агент** — автономна одиниця, що **сприймає** середовище через сенсори, **приймає рішення** й **впливає** на середовище через актуатори. Має мету (функцію корисності). Типи: прості рефлектори, з пам'яттю, цілеспрямовані, розумні, мультиагентні.
- **Середовище** — усе, що оточує агента й на що він впливає. Буває статичним (шахи) чи динамічним (реальний світ).
- **Задачі ШІ:** розпізнавання образів, обробка природної мови (NLP), машинне навчання, планування, робототехніка, ігровий ШІ, експертні та рекомендаційні системи.
- **Сильний ШІ (General AI)** — відтворює всі когнітивні здібності людини, самосвідомість (поки теоретичний). **Слабкий ШІ (Narrow AI)** — вузькоспеціалізований, для конкретних задач (Siri, рекомендації, розпізнавання) — найпоширеніший сьогодні.

## 10.2 Пошук у просторі станів і подання знань

### 10.2.1 Стратегії пошуку
- **Пошук вшир (BFS)** — рівень за рівнем через **чергу**; знаходить найкоротший шлях за кількістю кроків.
- **Пошук вглиб (DFS)** — максимально в глибину через **стек**, з відкатом.
- **Прямий пошук** — від початкового стану до цільового.
- **Зворотний пошук** — від цільового стану назад до початкового (напр. збірка кубика Рубіка від результату).
- **Двонаправлений пошук** — одночасно з обох боків; пошуки зустрічаються посередині, що зменшує кількість перевірених станів.

### 10.2.2 Моделі подання знань
- **Продукційна модель** — правила виду «**ЯКЩО** (умова/антецедент), **ТО** (дія/консеквент)»; висновок прямий (від даних до мети) або зворотний. Наочна, модульна; основа експертних систем (CLIPS, OPS5).
- **Семантична мережа** — орієнтований граф: вершини = поняття, дуги = відносини між ними (is-a/АКО, «має частину», «належить»). Недолік — складність організації логічного висновку.

## 10.3 Машинне навчання

### 10.3.1 Класифікація. Навчання з учителем/без учителя
**Задача класифікації** — віднесення об'єкта до одного з наперед заданих класів на основі навчальної вибірки.
- **Навчання з учителем (supervised)** — на **розмічених** даних (кожен приклад має мітку); задачі класифікації та регресії. Алгоритми: лінійна/логістична регресія, дерева рішень, KNN, SVM.
- **Навчання без учителя (unsupervised)** — на **нерозмічених** даних; пошук закономірностей: **кластеризація** (k-середніх), зниження розмірності (метод головних компонент), виявлення аномалій.
- **Напівкероване** — комбінація (мало розмічених + багато нерозмічених даних).
- **Навчання з підкріпленням (RL)** — через **винагороди/штрафи** від взаємодії із середовищем.

Основні типи задач: **класифікація** (категорії), **регресія** (числове прогнозування), **кластеризація** (групування без міток).

### 10.3.2 Тренувальні, валідаційні та тестові дані
- **Навчальний набір** — для підгонки (навчання) моделі.
- **Валідаційний набір** — для налаштування гіперпараметрів і вибору кращої моделі.
- **Тестовий набір** — для об'єктивної оцінки кінцевої моделі на невідомих даних.
Тестова вибірка має бути достатньо великою й репрезентативною. **Ніколи не навчатися на тестових даних** (веде до переоцінки якості). Альтернатива окремому набору — **k-кратна перехресна валідація**.

### 10.3.3 Штучний нейрон, ШНМ, функції активації
- **Штучний нейрон** (модель Маккалоха–Піттса) — вузол, що обчислює нелінійну **функцію активації** від зваженої суми входів. Типи: вхідні, вихідні, проміжні.
- **Штучна нейронна мережа (ШНМ)** — з'єднані нейрони, організовані в **шари** (вхідний → приховані → вихідний); ребра мають **ваги**, що підлаштовуються під час навчання.
- **Функції активації** (вводять нелінійність):
  - **Лінійна** — вихід пропорційний входу; для простих/регресійних моделей, не дає нелінійності.
  - **Порогова (східчаста)** — бінарна: 1, якщо вхід > порогу θ, інакше 0; для бінарної класифікації.
  - **Сигмоїдна** — S-подібна, вихід у діапазоні (0; 1); для прогнозування ймовірності, диференційована.
  - **Радіально-базисна (Гауса)** — залежить від відстані вхідного вектора до центру; значення тим більше, чим ближче до центру.

## Ключові факти для швидкого повторення

- **Складності:** O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ). Бінарний пошук — O(log n), швидке сортування — у середньому O(n log n).
- **BFS** = черга (FIFO), найкоротший шлях у незважених графах; **DFS** = стек (LIFO).
- **Стек = LIFO, черга = FIFO.**
- **Проблема зупинки** — класична **нерозв'язна** задача.
- **Доповнювальний код:** старший біт — знак, діапазон −2ⁿ⁻¹…2ⁿ⁻¹−1.
- **Фон Неймана** — спільна пам'ять; **Гарвардська** — роздільна.
- **Ієрархія пам'яті:** регістри → кеш → RAM → зовнішня.
- **Нормальні форми:** 1NF (атомарність) → 2NF (весь ключ) → 3NF (без транзитивних) → BCNF (детермінант = ключ).
- **ACID:** Atomicity, Consistency, Isolation, Durability.
- **SQL:** DDL (структура), DML (дані), DCL (доступ), TCL (транзакції).
- **4 принципи ООП:** абстракція, інкапсуляція, спадкування, поліморфізм.
- **Композиція** — частина не існує без цілого; **агрегація** — існує.
- **Флінн:** SISD, SIMD, MISD, MIMD.
- **Компілятор** — уся програма відразу (швидше); **інтерпретатор** — порядково (гнучкіше).
- **Верифікація** — «чи правильно побудували?»; **валідація** — «чи той продукт побудували?».
- **Emerging vs additive:** емерджентність — нові властивості цілого; адитивність — сума частин.
- **Навчання з учителем** — розмічені дані; **без учителя** — кластеризація.
- **Функції активації:** лінійна, порогова, сигмоїдна (0–1), радіально-базисна (Гауса).
